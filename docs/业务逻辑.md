# 业务逻辑

## 一、库存

### 1.1 库存查询（库存页面）

**数据来源**：`inventory` 表（仅 `quantity > 0` 的记录）

**状态判断**：
- **过期**：有效期日期 < 今天（空日期不判断，显示正常）
- **正常**：其他情况

**待审入库**：
- 来源：`in_order` status='pending' 的入库明细
- 匹配逻辑：按 `(material_id, batch_no)` 匹配已有库存行
  - 匹配到 → 在该行的 `pending_in` 字段累加数量
  - 匹配不到 → 新增一行，`quantity=0`，`pending_in=待审数量`，`status=正常`
- 同物料同批次的多条待审合并到同一行

**待审出库**：
- 来源：`out_order` status='pending' 的出库明细
- 匹配逻辑：按 `(material_id, batch_no)` 匹配已有库存行
- **不新增行**，只在已有库存行上累加 `pending_out`

**浮点精度**：数量运算使用 `ROUND(quantity, 2)`

### 1.2 库存更新

**入库审核时**：UPSERT on `(material_id, batch_no)`
- 批次已存在 → `quantity = quantity + ?`
- 批次不存在 → INSERT 新记录

**出库审核时**：扣减库存
- 指定批次 → 从该批次扣减（需库存足够）
- 未指定批次 → 按有效期升序扣减最早批次

**可回用物料**：出库审核时不扣减库存，退库审核时根据净用量扣减

### 1.3 库存导入

- 批次号为空时自动生成：`IMP-YYYYMMDDHHMMSS`
- 同一物料同一批次存在时更新数量，否则新增

## 二、订单

### 2.1 单号生成

| 订单类型 | 前缀 | 格式 |
|---------|------|------|
| 入库单 | RK | `RK-YYYYMMDD-序号`（当日自增） |
| 出库单 | CK | `CK-YYYYMMDD-序号`（当日自增） |
| 退库单 | TK | `TK-YYYYMMDD-序号`（当日自增） |

### 2.2 入库单

**状态**：`pending` → `approved`

**审核流程**：
1. 验证 status='pending'
2. 遍历明细，UPSERT inventory
3. 更新 status='approved'

**编辑/删除条件**：仅 `pending` 状态可操作

### 2.3 出库单

**状态**：`pending` → `approved` → `completed`

**审核流程**：
1. 验证 status='pending'
2. 遍历明细：
   - **可回用物料**：跳过库存扣减，创建称重记录
   - **普通物料**：按批次扣减库存
3. 更新 status='approved'

**编辑/删除条件**：仅 `pending` 状态可操作

### 2.4 退库单

**防重复**：同一出库单只能有一个 `approved` 状态的退库单

**审核流程**：
1. 验证无重复approved退库单
2. 更新 `reusable_material_weight` 称重记录（计算净用量）
3. 回冲库存：`inventory.quantity += 实际退库量`
4. 更新 `out_order.status = 'completed'`

**可回用物料净用量**：`initial_gross_weight - return_gross_weight`

## 三、物料

### 3.1 可回用物料

通过 `material.is_reusable = 1` 标识

**出库流程**：
- 不扣减库存
- 创建 `reusable_material_weight` 记录（status='checked_out'）

**退库流程**：
- 计算净用量：`initial - return`
- 扣减净用量对应的库存
- 更新 `reusable_material_weight`（status='returned'）

### 3.2 删除限制

有入库/出库记录 或 库存 > 0 → 禁止删除

## 五、报表

### 5.1 库存汇总报表

按物料分组，`quantity < safety_stock` → 显示"低于安全库存"

### 5.2 入库明细报表

按入库单明细展示，金额 = `数量 × 单价`

### 5.3 出库明细报表

按出库单明细展示，包含部门信息

## 六、数据库设计

### 6.1 单表设计

`inventory` 表以 `(material_id, batch_no)` 为 UNIQUE KEY，同一物料同一批次只存一条记录。

### 6.2 订单关联

| 表 | 关联 |
|---|------|
| in_order_item | order_id → in_order.id |
| out_order_item | order_id → out_order.id |
| inventory | material_id → material.id, in_order_item_id → in_order_item.id |
| return_order_item | return_order_id → return_order.id, out_order_item_id → out_order_item.id |
| reusable_material_weight | out_order_item_id → out_order_item.id |
